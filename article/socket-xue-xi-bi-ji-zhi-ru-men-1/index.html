    简单了解Socket

    Socket概述

    Socket是TCP/IP协议的编程模型，是提供两台计算机通信的网络编程接口，主要通过TCP/IP协议中的传输层的TCP,UDP协议的操作实体

    相关概念

    TCP（Transmission Control Protocol，传输控制协议）:面向连接的传输

    UDP（User Datagram Protocol，用户数据包协议）:面向连接

    简而言之，TCP是需要确认消息的传输，而UDP是只管发送，不需要存在连接

    IP （网络之间互连的协议）：主要区分局域网IP和公网IP

    Prot (端口)：用于区分计算机服务的一个访问手段

    Soket Demo分解

    C/S模式

    在TCP/IP网络应用中，通信的两个进程间相互作用的主要模式是客户/服务器（Client/Server, C/S）模式，即客户向服务器发出服务请求，服务器接收到请求后，提供相应的服务。

    通过Socket建立一个简单的客户端与服务端的通信。

    服务端的建立

    ps：本文采用Java作为例子，本文以TCP协议作为第一个Demo的小例子

    我们需要创建服务端

    Java提供了服务端的ServerSocket这个类，我们可以通过这个类去实现服务端的建立

    ServerSocket serverSocket = new ServerSocket(Prot); //建立绑定端口

    建立完就是开始监听客户端的消息

    使用ServerSocket中的accept()方法

    Socket client = serverSocket.accept();//等待客户端连接

    监听会一直等待到客户端的连接，当连接成功就进行下一步的消息处理

    如图客户1连接正在通信，此时一个新的用户的请求连接服务端未能及时响应，一直等待服务端的接收，直到客户1断开连接，服务端重新监听，才接收了客户2的连接，意味着上一个客户端连接如果没有断开，会一直占用服务端的处理

    本图为了实现多个客户端，我们一般通过开一个线程的方式去处理用户连接后的消息，服务端主要复制接收连接和创建线程去处理消息，这样就不会多客户的需要频繁等待，像上一个客户如果一直未响应就影响后续的客户的操作。

    多线程创建

    //这里采用嵌套静态子类，也可以单独创建一个外部类
private static class ClientHandler extends Thread{
        private Socket socket = null;
        private boolean flag = true;

        public ClientHandler(Socket socket) {
            this.socket = socket;
        }

        @Override
        public void run() {
            super.run();
            System.out.println("新客户端连接-ip:"+socket.getLocalAddress()+"/Port:"+socket.getPort());
            handleMsg(); //自定义处理消息函数，见后文
        }
    }

    Socket通信

    已经建立好了连接，那接下来就是客户端与服务端的通信过程

    我这里封装发送和接受消息的两个类

    //读取类
public class SoketInputUtil {
    private Socket socket = null;
    private BufferedReader soketReader = null;
    private InputStream inputStream = null;
    public SoketInputUtil(Socket socket) throws IOException {
        this.socket = socket;
        inputStream = socket.getInputStream();
        soketReader = new BufferedReader(new InputStreamReader(inputStream));
    }
    public String readMag() throws IOException {
        //从服务器接收一行
        return soketReader.readLine();
    }
    public void close(){//关闭资源 TODO    }
}

//发送消息类
public class SoketOutputUtil {
    private Socket socket = null;
    private PrintStream soketPrintStream = null;
    private OutputStream outputStream = null;
    public SoketOutputUtil(Socket socket) throws IOException {
        this.socket = socket;
        outputStream = socket.getOutputStream();
        soketPrintStream = new PrintStream(outputStream);
    }
    public Boolean sendMsg(String info)  {
        //发送到服务器
        soketPrintStream.println(info);
        return true;
    }
    public void close(){ //关闭资源 TODO }
}

    服务端和客户端处理消息的逻辑大致一致，为了方便简洁我这里写成函数

    //处理消息函数
private Boolean handleMsg() {
            SoketOutputUtil soketOutput = null;
            SoketInputUtil soketInput = null;
            try {
                do {
                //客户端消息代码可以从这个代码修改
                //=================================
                    //发送到客户端
                    soketOutput = new SoketOutputUtil(socket);
                    soketOutput.sendMsg("hello www.iflags.cn");
                    //从客户端接收一行
                    soketInput = new SoketInputUtil(socket);
                    String echo = soketInput.readMag();


                    //判断服务器是否断开连接
                    if ("bye".equalsIgnoreCase(echo)) {
                        flag = false;
                    } else { //打印服务器发送的信息
                        System.out.println(echo);
                        soketOutput.sendMsg("回送:"+echo.length());
                    }
                //=================================
                }while (false);
            } catch (IOException e) {
                e.printStackTrace();
            } finally {
                soketInput.close();
                soketOutput.close();
                System.out.println("断开连接");
            }
            //发送
            return true;
        }
